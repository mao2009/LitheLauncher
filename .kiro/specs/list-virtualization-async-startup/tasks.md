# Implementation Plan - list-virtualization-async-startup

リスト表示の仮想化と非同期処理を導入し、大量のデータ環境下でも0.7秒以内の高速起動と滑らかな操作性を実現します。

- [x] 1. 非同期データロード基盤の実装
- [x] 1.1 (P) バックグラウンドで全ゲーム情報を取得するワーカーの構築
  - 既存のゲームサービスを呼び出し、DBアクセスをバックグラウンド化する。
  - 結果をメインスレッドへ安全に渡すためのシグナルを定義する。
  - データ取得中のエラーハンドリングと通知機構を設ける。
  - _Requirements: 1.2_
- [x] 1.2* (P) 非同期ロードの検証テスト
  - ワーカーが正しくデータを取得し、シグナルを発火することを確認する。
  - DBアクセス失敗時の異常系ハンドリングをテストする。
  - _Requirements: 1.2_

- [x] 2. リスト仮想化エンジンの実装
- [x] 2.1 (P) スクロール位置に基づく表示範囲計算ロジックの構築
  - 表示領域（Viewport）のピクセル座標から、該当するゲームのインデックス範囲を算出する。
  - ウィンドウサイズ変更時に1行あたりの枚数を動的に再計算し、全体の仮想高さを更新する。
  - _Requirements: 2.1, 2.3_
- [x] 2.2 (P) ウィジェットの再利用（プーリング）機構の実装
  - 画面外に出たウィジェットをプールに回収し、新規表示分へ再割り当てする。
  - 既存ウィジェットの中身（データ）のみを高速に差し替える仕組みを構築する。
  - _Requirements: 2.1, 2.2_

- [x] 3. 高速起動のためのメインウィンドウ統合
- [x] 3.1 起動プロセスの変更とUIシェルの即時表示
  - アプリ起動時にまず空のメインウィンドウを即座に表示（0.7秒目標）するように変更する。
  - 表示完了の直後に非同期データロードを開始する。
  - _Requirements: 1.1, 1.3_
- [x] 3.2 仮想リストコントローラーの統合
  - スクロールエリアのスクロールイベントを仮想化エンジンに接続する。
  - データ取得完了時に仮想スクロール領域の全体サイズを確定させ、初期表示を行う。
  - _Requirements: 1.4, 2.1_

- [x] 4. 非同期画像表示の強化
- [x] 4.1 (P) 画像読み込みの優先順位付けと遅延実行
- [x] 4.2 (P) プレースホルダーと読み込み状態のUI改善


- [x] 5. 性能検証と最終調整
- [x] 5.1 1000件規模での起動速度とスクロール性能の最終計測
  - 大量データ投入環境で起動時間が0.7秒以内であることを実測する。
  - スクロール時のフレームレートが60fpsを維持しているか、カクつきがないか検証する。
  - _Requirements: 1.1, 2.3_
