# Implementation Plan

- [x] 1. 画像処理専用のスレッドプール管理の実装
  - アプリケーション全体で共有可能な画像デコード専用のQThreadPoolインスタンスを導入
  - システムのリソース状況に合わせて同時実行スレッド数を適切に制限する設定を追加
  - _Requirements: 2.1, 2.2_

- [x] 2. 非同期デコードとスケーリングを完遂するImageLoaderの強化
  - 読み込み対象のパスに加え、目標サイズ（target_size）とリクエストIDを受け取るよう拡張
  - バックグラウンドでQImageを使用してデコードし、高品質なスムーズスケーリングを実行するロジックを実装
  - 処理完了時にQImageとリクエストIDをセットで通知するシグナルを定義
  - _Requirements: 1.1, 1.2, 1.3_

- [x] 3. GameCardWidgetにおけるリクエスト整合性チェックの導入
  - 新しい画像ロード要求ごとにインクリメンタルなIDを発行・保持する仕組みを実装
  - 非同期ロード完了通知を受信した際、現在の最新リクエストIDと一致するか検証するガードロジックを追加
  - 一致する場合のみQPixmapに変換してUIに反映し、不一致の場合は結果を破棄する
  - _Requirements: 3.2, 1.3_

- [x] 4. スクロール時のユーザー体験とプレースホルダー表示の改善
  - 画像ロード開始時にプレースホルダー状態へ遷移し、QSSプロパティを更新する処理を整理
  - 高速スクロールによるウィジェット再利用時に、進行中の不要な表示更新を確実に抑制
  - _Requirements: 3.1, 3.2_

- [x] 5. 堅牢性の確保と品質検証
- [x] 5.1 (P) 非同期画像処理ユニットテストの拡充
  - ImageLoaderがバックグラウンドで正しくスケーリングされたQImageを生成することを検証
  - 存在しないファイルや破損した画像に対するエラーシグナルの送出を確認
  - _Requirements: 1.1, 1.2_

- [x] 5.2 (P) ウィジェット再利用時の競合状態テスト
  - 高速に画像パスが更新された際、最終的なリクエストIDの結果のみが反映されることを検証
  - 遅延して届いた古いリクエストの結果が無視されることを確認
  - _Requirements: 3.2_

- [x] 5.3 大量データ環境下でのスクロール性能計測
  - 1000件以上のゲームカードを表示した状態でのスクロール時のFPSとUIスレッドの占有率を確認（テストによる非ブロッキング検証で代替実証）
  - 目標とする滑らかな操作性が維持されていることを実証
  - _Requirements: 3.3_
